# DESIGN.md — TalavaPlayer デザインシステム

スクリーンショット（動画グリッド / サイドバー / 再生履歴 / 再生リスト）から抽出した
ダークテーマ動画プレイヤー UI のデザインシステム。

トークンは 2 層構成：

- **プリミティブトークン (Primitive)** — 色・寸法・字形などの「見た目そのもの」を表す生の値。意味を持たない。
- **セマンティックトークン (Semantic)** — プリミティブを 1 個以上組み合わせ、用途・役割で意味付けした参照。UI 実装はこちらを参照する。

> 命名規約: プリミティブは `--p-…`、セマンティックは `--s-…`。実装言語に合わせて読み替え可。

## 1. 観察された UI 要素（抽出元）

| 画面 | 抽出した要素 |
|------|--------------|
| 動画グリッド | 4 カラムのビデオカード、16:9 サムネイル、再生時間バッジ、LIVE バッジ、チャンネルアバター、2 行タイトル、メタ情報、ケバブメニュー、「もっと見る」ピルボタン |
| サイドバー | アイコン+ラベルのナビ行、セクション見出し、小アバター+ライブ/通知ドット、選択中行のハイライト |

| 再生リスト | ページ見出し、ソートドロップダウン、スタック表現のサムネイル、件数オーバーレイバッジ、補助リンク |

## 4. コンポーネント仕様

各コンポーネントはセマンティックトークンのみで定義する。

### 4.1 ビデオカード (VideoCard)

```
┌─────────────────────────┐
│  [サムネイル 16:9]       │  ← --s-radius-container
│              ⌛1:11:13    │  ← 時間バッジ（右下）
│  🔴 ライブ                │  ← LIVE バッジ（左下、ライブ時のみ）
└─────────────────────────┘
( ◯ ) タイトル行1            ⋮  ← アバター(channel) / タイトル / ケバブ
       タイトル行2
       チャンネル名 ✓
       4.3万回視聴・11 分前
```

- 構造: サムネイル + 本文ブロック（縦積み）。
- 本文は `アバター(--s-size-avatar-channel)` + テキスト列を `--s-space-gap-tight` で横並び。
- タイトル: `--s-type-card-title`、2 行でクランプ（3 点リーカット）。
- チャンネル名: `--s-type-channel` + 認証チェック `--s-icon-verified`。
- メタ: `--s-type-meta`。区切りは中黒「・」。
- ケバブメニュー: `--s-icon-muted`、ホバーで `--s-bg-hover`。
- サムネ右下「時間バッジ」: 地 `--s-bg-scrim` / 文字 `--s-type-badge` / `--s-radius-overlay`。

### 4.2 LIVE バッジ (LiveBadge)

- 地 `--s-accent-live` / 文字 `--s-text-on-accent` / `--s-radius-overlay`。
- ラベル「ライブ」、必要に応じ先頭に白丸/放送アイコン。
- 配置: サムネ左下、`--s-space-overlay-offset` のオフセット。

### 4.3 ピルボタン (PillButton) — 「もっと見る」

- 地 `--s-bg-elevated` / 文字 `--s-text-primary` / `--s-radius-control-pill`。
- 左右 `--s-space-inset-pill`、上下 `--s-space-overlay-offset`。末尾に開閉キャレット `∨`。
- ホバー: `--s-bg-hover`。

### 4.4 フィルタチップ (FilterChip)

| 状態 | 地 | 文字 |
|------|-----|------|
| 非選択 | `--s-bg-elevated` | `--s-text-primary` |
| 選択 | `--s-bg-inverse` | `--s-text-inverse`（反転）|

- 角丸 `--s-radius-control-soft`、パディング `--s-space-inset`。
- 例: `すべて / 動画 / ショート / ポッドキャスト / 音楽`。

### 4.5 サイドバー行 (NavItem)

```
[icon] ラベル                         ← 高さ --s-size-nav-row
```

- アイコン `--s-icon-default`（`--s-size-icon`）+ ラベル `--s-type-body`。
- アイコン↔ラベル間 `--s-space-gap-tight`、行パディング `--s-space-inset`。
- ホバー: 角丸 `--s-radius-control-soft` の `--s-bg-hover`。
- 選択中: `--s-bg-selected`。
- セクション見出し（「登録チャンネル」「マイページ」）: `--s-type-meta` 相当、`>` 付き。

### 4.6 チャンネル行（サイドバー内）

- 小アバター `--s-size-avatar-nav`（`--s-radius-circle`）+ チャンネル名 `--s-type-body`。
- 状態ドット `--s-size-indicator`（`--s-radius-circle`）:
  - 配信中 → `--s-accent-brand`（赤）。
  - 新着通知 → `--s-indicator-notify`。

### 4.7 リストアイテム（再生履歴 RowItem）

- 横並び: 大サムネ（左、16:9、`--s-radius-container`、時間バッジ付き）+ テキスト列。
- テキスト列: タイトル `--s-type-card-title` → チャンネル `--s-type-channel` → メタ `--s-type-meta`。
- 右端にケバブ `--s-icon-muted`。
- 日付セクション見出し（「今日」「昨日」）: `--s-type-section`。

### 4.8 プレイリストカード (PlaylistCard)

- スタック表現のサムネ（背面に薄いカードを重ねた多重表現）。
- 件数オーバーレイ（右下）: 地 `--s-bg-scrim` / 文字 `--s-type-badge`、リストアイコン + 「100 本の動画」。
- 下部: タイトル `--s-type-card-title` → 「非公開・プレイリスト」`--s-type-meta` → 「本日更新」`--s-type-meta` → 「再生リストの全体を見る」リンク（`--s-text-secondary`、ホバーで `--s-text-primary`）。

## 5. レイアウト原則

- **グリッド**: 動画一覧は `--s-grid-columns`（既定 4）カラム、間隔 `--s-space-gap-loose`。幅に応じ列数を可変。
- **アスペクト比**: サムネイルは常に `--s-ratio-media`（16:9）を維持。
- **階層**: 地 `--s-bg-canvas` → 面 `--s-bg-surface` → 浮き `--s-bg-elevated` の 3 段で前後関係を表現（影は基本使わず、明度差で表現）。
- **コントラスト**: 主テキストは `--s-text-primary`、補助は必ず `--s-text-secondary` に落として情報階層を作る。
- **アクセントは赤のみ**: 装飾色を増やさず、LIVE/通知/ブランドに限定して赤を使う。

## 6. 利用ガイドライン

- 実装では **セマンティックトークンのみ** を参照する。プリミティブを直接埋め込まない（テーマ差し替え・調整の起点を一本化するため）。
- 新しい色・寸法が必要になったら、まずプリミティブに 1 件追加 → それを指すセマンティックを定義、の順で拡張する。
- 値はスクリーンショットからの実測・推定。実装後はキャプチャ検証（`--enable-dev-tools` の `/screenshot`）で突き合わせて微調整する。
