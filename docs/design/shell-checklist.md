# shell 変更時の手動チェックリスト

対象読者: `src/ui/shell.rs`（winit 配線 + Win32 の暗黙知）を触る人。

`shell.rs` は Issue #11 PR U で「地雷（暗黙知）」を集約する場所として切り出した。以後ほぼ不変を
意図しているが、やむを得ず変更する場合は次の3件を手動で確認すること（自動テストでは検知できない、
過去に実際デグレした挙動）。

## 1. hwdec 切替中の自動非表示

`WindowEvent::Resized` ハンドラで `last_activity` を更新してはいけない。hwdec（HW/SW デコード）切替は
mpv の VO 再初期化に伴いプログラム的に `Resized` を連発するため、これを操作扱いすると
オーバーレイの自動非表示（3秒無操作で隠れる）が効かなくなる。

**確認方法**: 画質切替や GPU 負荷変動で hwdec が自動切替されている間（`[auto-hwdec]` ログを参照）、
オーバーレイ帯が正しく自動的に隠れることを目視確認する。

## 2. ウィンドウドラッグ中のオーバーレイ追従

`WindowEvent::Resized` でオーバーレイをリサイズする処理は、モーダルなドラッグループ中は
`about_to_wait` が呼ばれない winit の制約に対応するためのもの。ドラッグ中も追従させる必要がある処理は
`window_event` 側（イベント直下）に置く。

**確認方法**: ウィンドウの端をドラッグしてリサイズしている最中も、コントローラ帯・一覧・チャットが
瞬時に追従することを目視確認する。

## 3. フォーカス喪失時のチャット表示継続

`WindowEvent::Focused(false)` でオーバーレイを隠してはいけない。非アクティブウィンドウでも
チャット等を表示し続ける設計（他アプリを操作しながら視聴する用途を想定）。可視判定は
`about_to_wait` の `active` 判定（無操作 3 秒 or 一覧/チャット表示中）に一任する。

**確認方法**: 動画ウィンドウからフォーカスを外した（別アプリをクリックした）状態でも、
チャットパネルが表示されたままであることを目視確認する。
